# ネット販売出品者支援Chrome拡張（仮称）計画書

## 背景と課題
- eBayでは約1,760万のアクティブセラー、1.34億のアクティブバイヤー、Etsyでは約560万のアクティブセラーが存在し、膨大な潜在ユーザーがいる。
- 多くのセラーが副業や趣味として運営し、タイトル・説明文作成、カテゴリ選定、価格調査、在庫調整、セール反映などの反復作業に時間を取られている。
- 古いリスティング管理や一括更新、価格自動化などが不足しているという声があり、AIや自動化による工数削減ニーズが高い。

## ターゲットユーザー
- Etsy、eBay、Shopify、Amazonハンドメイド等で数十〜数百商品を扱う個人・小規模セラー。
- 特にEtsyでは副業セラーが多く、手作業削減への関心が強い。コレクションアイテムやハンドメイドなど専門性が高いカテゴリのセラーも含む。

## 提案機能
1. **AI生成支援**: 商品画像や既存情報からSEO最適化タイトル・説明文、カテゴリ提案を自動生成。
2. **価格・在庫の自動提案/更新**: 類似商品や販売履歴を分析して最適価格を提示し、在庫切れや再出品を自動化。
3. **一括操作・データ連携**: CSVアップロードで複数商品の一括出品・更新、売上データのCSVエクスポートで会計ソフト連携を支援。
4. **マルチプラットフォーム連携**: Amazon、Shopify、メルカリ、ポッシュマーク等へのクロスリスティング。
5. **分析・レポート**: カテゴリ別売上傾向、キーワード分析、需要予測や利益率推定を提供。

## 収益化戦略
- 月額サブスクリプション（約$5〜$15）を想定。複数プランで機能・出品数上限・サポート水準を差別化し、アップセルを狙う。
- 数千万〜数百万規模の潜在セラーの1〜数%が課金しても成立する市場規模。

## 競合・差別化
- 既存SaaSにはNifty、ZIK Analytics、3Dsellers、InkFrog、Vendoo、ListPerfectlyなどがあるが、Chrome拡張でブラウザ上に直接埋め込む総合ツールは少ない。
- 優位性: ページDOMを直接読み取り即時AI提案、画像認識＋在庫履歴反映による生成精度、Amazon/Shopifyまで広げたクロスリスティング、QuickBooks/Xero等との会計連携、特定カテゴリに特化した分析。

## 開発アプローチ
1. **MVP構築**: 無料βとしてタイトル生成・一括更新を先行実装しユーザーテスト。
2. **フィードバック反映**: セラーの要望を継続収集し優先度順に改善。
3. **AIエージェント活用**: Copilot等でmanifest/popup/contentスクリプトの自動生成、デバッグ、ドキュメンテーション効率化。
4. **スケーラビリティ検証**: 必要に応じてサーバーレスAPIを用意し、ユーザー増加時の応答性を確保。
5. **運用改善**: 行動データを継続学習に活用し、生成精度と自動化ロジックを強化。
